**Page structure:**

1. **Page intro** (`<!-- ai:crud-operations -->`)
   - Brief explanation: DataStore conflates create/update into `save()` and handles `_version` internally. Apollo separates these into distinct mutations and requires explicit `_version` handling.
   - Link to Prerequisites page for GraphQL operation definitions (POST_DETAILS_FRAGMENT, CREATE_POST, UPDATE_POST, DELETE_POST, GET_POST, LIST_POSTS).

2. **Create (Save New Record)** (`<!-- ai:crud-create -->`) -- CRUD-01
   - Explain: `new Model()` + `DataStore.save()` = CREATE mutation
   - Before: `DataStore.save(new Post({ title, content, status, rating }))`
   - After (imperative): `apolloClient.mutate({ mutation: CREATE_POST, variables: { input: { title, content, status, rating } } })`
   - After (React hook): `useMutation(CREATE_POST, { refetchQueries: [{ query: LIST_POSTS }] })`
   - Note: No `_version` needed for create -- AppSync sets it to 1 automatically
   - Use `<!-- before: DataStore -->` and `<!-- after: Apollo Client -->` labels per CONTEXT.md decision

3. **Update (Modify Existing Record)** (`<!-- ai:crud-update -->`) -- CRUD-02
   - Explain: `Model.copyOf()` uses immer for immutable updates -> Apollo uses plain objects. Only changed fields needed in input (partial update). `_version` is REQUIRED.
   - Before: `Post.copyOf(original, draft => { draft.title = 'Updated' })` + `DataStore.save()`
   - After (imperative): query first to get `_version`, then `apolloClient.mutate({ mutation: UPDATE_POST, variables: { input: { id, title, _version } } })`
   - After (React hook): `useQuery(GET_POST)` for current data + `useMutation(UPDATE_POST)` with `_version` from query result
   - Warning callout: "If you see `ConditionalCheckFailedException`, you are missing or passing a stale `_version`"

4. **Delete (Single Record)** (`<!-- ai:crud-delete -->`) -- CRUD-03
   - Explain: DataStore.delete() had 3 overloads (by instance, by ID, by predicate). Apollo only supports single-record delete.
   - Before: `DataStore.delete(postInstance)` or `DataStore.delete(Post, '123')`
   - After: Query first to get `_version`, then `apolloClient.mutate({ mutation: DELETE_POST, variables: { input: { id, _version } } })`
   - Warning: Must query first to get current `_version` even for delete

5. **Query by ID** (`<!-- ai:crud-query-id -->`) -- CRUD-04
   - Before: `DataStore.query(Post, '123')`
   - After (imperative): `apolloClient.query({ query: GET_POST, variables: { id: '123' } })`
   - After (React hook): `useQuery(GET_POST, { variables: { id: '123' } })`
   - Note: Apollo returns `null` for non-existent IDs (DataStore returns `undefined`)

6. **List All Records** (`<!-- ai:crud-list -->`) -- CRUD-05
   - Before: `DataStore.query(Post)`
   - After (imperative): `apolloClient.query({ query: LIST_POSTS })`
   - After (React hook): `useQuery(LIST_POSTS)`
   - Critical: Must filter `_deleted` records: `.items.filter(p => !p._deleted)`
   - Show the `filterDeleted` helper function from prerequisites

7. **Batch Delete (Predicate-Based)** (`<!-- ai:crud-batch-delete -->`) -- CRUD-10
   - Explain: DataStore supported `DataStore.delete(Post, p => p.status.eq('DRAFT'))`. Apollo has no equivalent -- must query matching records then delete each individually.
   - Before: `DataStore.delete(Post, p => p.status.eq('DRAFT'))`
   - After: Query with filter, then `Promise.all()` to delete each record with its `_version`
   - Warn about: no atomicity (some deletes may succeed and others fail), rate limiting on large batches
   - Show error handling pattern for partial batch failures

8. **Quick Reference Table** (`<!-- ai:crud-reference -->`)
   - Summary table: DataStore method -> Apollo equivalent -> Key difference
   - Cover all 6 operations in one scannable table

9. **Common Mistakes** (`<!-- ai:crud-mistakes -->`)

   - Using CREATE mutation for updates (DataStore `save()` handled both)
   - Not filtering `_deleted` records from list results
   - Not using `refetchQueries` after mutations (stale list)
