1. **Page intro** (`<!-- ai:relationships -->`)
   - Core concept: DataStore lazy-loads relationships via Promises and AsyncCollections. You access a field and it fetches on demand. Apollo eagerly loads relationships based on what you include in your GraphQL selection set. This gives you explicit control over data fetching granularity but requires you to think about what data you need upfront.
   - Show the Amplify Gen 2 schema definitions used in examples:
     ```typescript
     // amplify/data/resource.ts (relevant models)
     const schema = a.schema({
       Post: a.model({
         title: a.string().required(),
         content: a.string(),
         status: a.string(),
         rating: a.integer(),
         comments: a.hasMany('Comment', 'postId'),
         tags: a.hasMany('PostTag', 'postId'),
         metadata: a.hasOne('PostMetadata', 'postId'),
       }),
       Comment: a.model({
         content: a.string().required(),
         postId: a.id().required(),
         post: a.belongsTo('Post', 'postId'),
       }),
       Tag: a.model({
         name: a.string().required(),
         posts: a.hasMany('PostTag', 'tagId'),
       }),
       PostTag: a.model({
         postId: a.id().required(),
         tagId: a.id().required(),
         post: a.belongsTo('Post', 'postId'),
         tag: a.belongsTo('Tag', 'tagId'),
       }),
       PostMetadata: a.model({
         postId: a.id().required(),
         views: a.integer(),
         likes: a.integer(),
         post: a.belongsTo('Post', 'postId'),
       }),
     });
     ```
   - Note: All relationship examples include `_version`, `_deleted`, `_lastChangedAt` fields in selections for conflict-resolution-enabled backends.

2. **hasMany: Post -> Comments** (`<!-- ai:has-many -->`) -- RELS-01
   - Explain: DataStore's `AsyncCollection` with `.toArray()` becomes a nested GraphQL selection with `items` wrapper.
   - Before:
     ```typescript
     const post = await DataStore.query(Post, '123');
     const comments = await post.comments.toArray();
     ```
   - After -- GraphQL query definition:
     ```graphql
     query GetPostWithComments($id: ID!) {
       getPost(id: $id) {
         ...PostDetails
         comments {
           items {
             id
             content
             createdAt
             _version
             _deleted
             _lastChangedAt
           }
           nextToken
         }
       }
     }
     ```
   - After -- Apollo usage (imperative and React hook):
     ```typescript
     const { data } = await apolloClient.query({
       query: GET_POST_WITH_COMMENTS,
       variables: { id: '123' },
     });
     const comments = data.getPost.comments.items.filter(c => !c._deleted);
     ```
   - Key insight: DataStore lazy-loaded (fetch on access). Apollo eager-loads (fetch with parent). To simulate lazy loading, omit `comments` from the initial query and make a separate query when needed.
   - Show the separate-query (lazy) pattern as an alternative:
     ```graphql
     query ListCommentsByPost($filter: ModelCommentFilterInput) {
       listComments(filter: $filter) {
         items { id content createdAt _version _deleted }
         nextToken
       }
     }
     ```
     With `variables: { filter: { postId: { eq: '123' } } }`
   - Warn about over-fetching: including relationships in every query when the component doesn't always need them.

3. **belongsTo: Comment -> Post** (`<!-- ai:belongs-to -->`) -- RELS-02
   - Explain: DataStore resolves the parent automatically via Promise. Apollo uses nested selection.
   - Before:
     ```typescript
     const comment = await DataStore.query(Comment, 'abc');
     const post = await comment.post; // Promise resolves to Post
     ```
   - After -- GraphQL query:
     ```graphql
     query GetCommentWithPost($id: ID!) {
       getComment(id: $id) {
         id
         content
         post {
           id
           title
           status
           _version
         }
         _version
         _deleted
         _lastChangedAt
       }
     }
     ```
   - After -- usage: `data.getComment.post` gives the parent directly.
   - Note: The foreign key field (`postId`) is also available on the Comment if you only need the ID without fetching the full parent.

4. **hasOne: Post -> PostMetadata** (`<!-- ai:has-one -->`) -- RELS-03
   - Explain: Similar to belongsTo but represents a 1:1 ownership. DataStore returns a Promise, Apollo uses nested selection.
   - Before:
     ```typescript
     const post = await DataStore.query(Post, '123');
     const metadata = await post.metadata; // Promise resolves
     ```
   - After -- GraphQL query:
     ```graphql
     query GetPostWithMetadata($id: ID!) {
       getPost(id: $id) {
         ...PostDetails
         metadata {
           id
           views
           likes
           _version
         }
       }
     }
     ```
   - After -- usage: `data.getPost.metadata` gives the metadata (or `null` if none).

5. **manyToMany: Post <-> Tag** (`<!-- ai:many-to-many -->`) -- RELS-04
   - Explain: Many-to-many relationships use an implicit join table model (PostTag). DataStore auto-generates this. In GraphQL, you must query through the join model explicitly.
   - Before:
     ```typescript
     const post = await DataStore.query(Post, '123');
     const postTags = await post.tags.toArray(); // returns PostTag join records
     // Each PostTag has .tag which resolves to the Tag
     const tags = await Promise.all(postTags.map(pt => pt.tag));
     ```
   - After -- GraphQL query:
     ```graphql
     query GetPostWithTags($id: ID!) {
       getPost(id: $id) {
         ...PostDetails
         tags {
           items {
             id
             tag {
               id
               name
               _version
             }
             _version
             _deleted
           }
         }
       }
     }
     ```
   - After -- extracting tags:
     ```typescript
     const tags = data.getPost.tags.items
       .filter(pt => !pt._deleted)
       .map(pt => pt.tag);
     ```
   - Show how to create a many-to-many association (create a PostTag record):
     ```typescript
     await apolloClient.mutate({
       mutation: CREATE_POST_TAG,
       variables: { input: { postId: '123', tagId: '456' } },
     });
     ```
   - Show how to remove a many-to-many association (delete the PostTag record).

   - Eager vs lazy loading tradeoffs
